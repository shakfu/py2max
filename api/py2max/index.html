    py2max Package¶

    The top-level py2max package exports the main classes for creating Max/MSP patcher files.

    Main classes¶

    • Patcher -- create and manipulate Max patches
    • Box -- an individual Max object in a patch
    • Patchline -- a connection between objects
    from py2max import Patcher

p = Patcher('example.maxpat')
osc = p.add_textbox('cycle~ 440')
gain = p.add_textbox('gain~ 0.5')
dac = p.add_textbox('ezdac~')
p.add_line(osc, gain)
p.add_line(gain, dac)
p.to_svg('example.svg')   # preview
p.save()

    The SQLite Max-reference database is available lazily as from py2max.maxref import MaxRefDB (kept off the top-level import so import py2max stays light and dependency-free).

    Exceptions¶

    Exception hierarchy for py2max.

    This module defines a comprehensive exception hierarchy for error handling throughout the py2max library. All exceptions inherit from Py2MaxError for easy catching of library-specific errors.

    Exception Hierarchy

    Py2MaxError (base) ├── ValidationError │ ├── InvalidConnectionError (connections between objects) │ ├── InvalidObjectError (invalid object configuration) │ └── InvalidPatchError (invalid patcher state) ├── ConfigurationError │ ├── LayoutError (layout manager issues) │ └── DatabaseError (database configuration issues) ├── IOError (subclass of built-in IOError) │ ├── PatcherIOError (file I/O errors) │ └── MaxRefError (maxref.xml parsing errors) └── InternalError (unexpected internal errors)

    Example

    from py2max.exceptions import InvalidConnectionError try: ... p.add_line(osc, gain, outlet=999) ... except InvalidConnectionError as e: ... print(f"Connection error: {e}")

    Py2MaxError ¶

    Bases: Exception

    Base exception for all py2max errors.

    All py2max-specific exceptions inherit from this class, allowing users to catch all library errors with a single exception handler.

    Attributes:

    Name Type Description
    message

    Error message describing what went wrong.
    context

    Optional dict with additional error context.

    ValidationError ¶

    Bases: Py2MaxError

    Base class for validation errors.

    Raised when user input or object state fails validation checks.

    InvalidConnectionError ¶

    Bases: ValidationError

    Raised when attempting to create an invalid patchline connection.

    This exception is raised when validation is enabled and an invalid connection is attempted, such as: - Connecting to a non-existent inlet or outlet - Connecting incompatible signal types - Creating duplicate connections

    Attributes:

    Name Type Description
    src

    Source object identifier.
    dst

    Destination object identifier.
    outlet

    Source outlet index (if applicable).
    inlet

    Destination inlet index (if applicable).

    InvalidObjectError ¶

    Bases: ValidationError

    Raised when object configuration is invalid.

    Examples: - Invalid object type name - Missing required parameters - Invalid parameter values - Unknown maxclass

    Attributes:

    Name Type Description
    object_id

    Object identifier.
    maxclass

    Max object class name.

    InvalidPatchError ¶

    Bases: ValidationError

    Raised when patcher state is invalid.

    Examples: - Circular subpatcher references - Missing required objects - Orphaned connections - Invalid patch structure

    Attributes:

    Name Type Description
    patch_path

    Path to the patcher file.

    ConfigurationError ¶

    Bases: Py2MaxError

    Base class for configuration errors.

    Raised when library configuration is invalid or incompatible.

    LayoutError ¶

    Bases: ConfigurationError

    Raised when layout manager encounters an error.

    Examples: - Unknown layout type - Invalid layout parameters - Layout algorithm failure

    Attributes:

    Name Type Description
    layout_type

    Layout manager type.

    DatabaseError ¶

    Bases: ConfigurationError

    Raised when database operations fail.

    Examples: - Database connection failure - Schema mismatch - Data corruption - Query errors

    Attributes:

    Name Type Description
    db_path

    Path to database file.
    operation

    Operation that failed.

    PatcherIOError ¶

    Bases: Py2MaxError, IOError

    Raised when patcher file I/O operations fail.

    Examples: - File not found - Permission denied - Invalid JSON format - Disk full

    Attributes:

    Name Type Description
    file_path

    Path to the file.
    operation

    I/O operation that failed.

    MaxRefError ¶

    Bases: Py2MaxError, IOError

    Raised when MaxRef XML parsing or lookup fails.

    Examples: - MaxRef XML file not found - Invalid XML format - Object not found in MaxRef database - Max installation not found

    Attributes:

    Name Type Description
    object_name

    Max object name being looked up.
    xml_path

    Path to .maxref.xml file.

    InternalError ¶

    Bases: Py2MaxError

    Raised for unexpected internal errors.

    This exception indicates a bug in the library code and should be reported. Users should not typically need to catch this exception.

    Attributes:

    Name Type Description
    location

    Location in code where error occurred.
